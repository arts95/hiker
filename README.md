# Алгоритм розподілу чергувань (Camp Duty Scheduler)

Цей скрипт призначений для автоматичного створення максимально справедливого розкладу чергувань на кухні для туристичного табору.

## Як запустити
1. Переконайтеся, що у папці зі скриптом лежить файл `data.csv` (експортований з Google Таблиць).
2. Запустіть скрипт командою:
   ```bash
   python3 duty_scheduler.py
   ```
3. Скрипт згенерує файл `schedule_output.csv`, який ви можете завантажити назад у Google Таблиці (Файл -> Імпортувати).

---

## 🧠 Як працює алгоритм?

Алгоритм базується на принципі **"Динамічного боргу" (Жадібний алгоритм з пріоритетною чергою)**. 

### 1. Оцінка прийомів їжі (Бали)
Кожен прийом їжі має свою вагу (складність). Наприклад:
* **Сніданок:** 1.0 бал
* **Обід:** 1.0 бал
* **Вечеря:** 1.0 бал

### 1.1 Межі табору
Ви можете налаштувати, з якого прийому їжі починається табір у перший день, і яким закінчується в останній:
* `CAMP_START_MEAL = "Обід"` (наприклад, у день заїзду сніданку ще немає)
* `CAMP_END_MEAL = "Сніданок"` (наприклад, у день роз'їзду обіду і вечері вже немає)
Ці параметри легко змінити на початку файлу `duty_scheduler.py`.

### 1.2 Ручне фіксування чергувань (FIXED_DUTIES)
Якщо вам потрібно примусово призначити певне чергування (наприклад, якась група попросила вечерю в конкретний день), ви можете вказати це у словнику `FIXED_DUTIES` на початку скрипту:
```python
FIXED_DUTIES = {
    # Формат: "ДД.ММ": {"Прийом їжі": "Назва групи"}
    "19.07": {"Вечеря": "Екіпаж 1 (Аліса/Боб/Влад)"}
}
```
Алгоритм автоматично:
1. Примусово поставить цю групу на чергування у вказаний час.
2. Подбає про те, щоб ця група відпочивала до і після зафіксованого прийому їжі (запобігає чергуванням два рази поспіль).
3. Збалансує решту розкладу, щоб інші групи перекрили ті зміни, які звільнилися.


### 2. Нарахування "Боргу" (Fair Share) та формат "Дорослі + Діти"
Замість того, щоб рахувати просто кількість разів, алгоритм рахує справедливу частку роботи в балах. 
Борг нараховується **лише тим групам, які фізично присутні в таборі** в конкретний день. 

У стовпчику кількості людей підтримується формат **`X+Y`** (наприклад, `2+1` — двоє дорослих та одна дитина):
* **Коефіцієнт споживання (Мета):** При розрахунку порцій дитина споживає менше ніж дорослий (за замовчуванням `KID_MEAL_WEIGHT = 0.5` порції дорослого). Тобто група `2+1` накопичуватиме борг як `2.5` особи.
* **Коефіцієнт роботи (Чергування):** У робочу силу чергової бригади зараховуються **лише дорослі** (`work_size`).
> *Якщо група приїхала на 3 дні пізніше, вона починає з нульового боргу і не має "відпрацьовувати" за ті дні, коли її не було.*

### 3. Процес призначення чергових
Алгоритм йде хронологічно від першого сніданку до останньої вечері. Для кожного прийому їжі він виконує такі кроки:

1. **Фільтрація:** Відкидає всі групи, яких зараз немає в таборі.
2. **Відпочинок:** Відкидає групи, які готували зовсім недавно. Діє правило календарного відпочинку (`MIN_REST_DAYS = 1`), що забороняє одній групі чергувати на сусідні (послідовні) календарні дні. Тобто, якщо група чергувала в понеділок, наступне чергування можливе щонайменше в середу. Якщо при дуже жорстких обмеженнях кандидатів не знайдено, система автоматично робить безпечний відкат до меншого відпочинку.
3. **Сортування (Вибір найнесправедливішого):** Серед тих, хто доступний, алгоритм обирає групу, у якої показник `(Борг - Відпрацьовані бали)` є найбільшим. Тобто ту групу, яка на даний момент найбільше "винна" табору.
   * **Штраф за одноманітність (Diversity Penalty):** Щоб уникнути ситуації, коли одна група готує тільки сніданки, алгоритм штрафує (занижує пріоритет) групу на конкретний прийом їжі, якщо вона вже готувала його в минулому. Цей штраф дорівнює `1.0` бал, що дозволяє оптимально балансувати різноманітність меню та закриття боргів (щоб групи не їхали з недовиконанням норми).
4. **Об'єднання одинаків:**
   * Якщо в обраній групі **>1 дорослого** (наприклад, `2`, `2+1` чи `3`), вона готує САМА (їй нікого не дають у допомогу, щоб не створювати натовп на кухні).
   * Якщо в обраній групі **1 дорослий** (одинак, наприклад, `1` чи `1+2`), алгоритм шукає їй у пару **іншу групу з 1 дорослим**.
5. **Оплата боргу:** Коли група чергує, її "Відпрацьовані бали" збільшуються на вагу цього прийому їжі. Якщо готували дві групи з 1 дорослого кожна, вони ділять ці бали порівну.

## 📊 Що містить вихідний файл?
Згенерований файл `schedule_output.csv` містить 3 блоки:
1. **Лінійний розклад** (Дата | Прийом їжі | Хто чергує).
2. **Статистика справедливості** — таблиця, яка показує, наскільки фактично відпрацьовані бали відрізняються від ідеального математичного боргу (Різниця зазвичай не перевищує 1 балу, що є похибкою менше ніж в один сніданок).
3. **Матриця чергувань** — зручна таблиця-шахівниця (рядки — групи, стовпчики — дати), де позначено, хто готує Сніданок (С), Обід (О) чи Вечерю (В), а також прочерки (-) для днів відсутності в таборі.
